# OpenClaw

OpenClaw（原名 Clawdbot / Moltbot）是一个运行在你自己设备上的**多渠道 AI 网关**。它可以在你已经使用的各种消息平台上回复你，包括 Telegram、WhatsApp、Slack、Discord、iMessage、Signal 等，让你的 AI 助手无处不在。

<Callout type="info">
  **开源项目**：OpenClaw 是完全开源的，你可以在 [GitHub](https://github.com/openclaw/openclaw) 上查看源码、提交 Issue 或贡献代码。
</Callout>

## 核心优势

### 1. 本地优先，隐私可控
OpenClaw 运行在你自己的设备上，所有数据都在本地处理：
- **完全掌控**：你的对话数据不会上传到第三方服务器
- **本地 Gateway**：单一控制平面管理所有会话、频道和工具
- **自定义代理**：支持配置第三方 API 代理（如 CC Club），无需翻墙

### 2. 多渠道消息支持
OpenClaw 支持 20+ 主流消息平台：
- **即时通讯**：WhatsApp、Telegram、Signal、iMessage
- **团队协作**：Slack、Discord、Microsoft Teams、Google Chat
- **扩展支持**：Matrix、Zalo、WebChat 等
- **统一体验**：在任何平台上都能获得一致的 AI 助手体验

### 3. 多模型支持
支持多种 AI 模型后端：
- **Anthropic Claude**：通过 CC Club 代理或官方 API
- **OpenAI**：GPT 系列模型
- **Google**：Gemini 系列
- **本地模型**：通过 Ollama 运行本地模型
- **模型切换**：可以根据任务需求灵活切换模型

### 4. 语音交互能力
支持在多个平台上进行语音交互：
- **Voice Wake**：始终在线的语音唤醒功能
- **Talk Mode**：连续对话模式，自然的语音交互体验
- **跨平台支持**：macOS、iOS、Android 全平台语音支持

### 5. 可扩展的技能系统
通过自定义技能（Skills）扩展功能：
- **提示词模板**：可复用的自定义提示词
- **定时任务**：通过 Cron 管理定期提醒
- **工具生态**：浏览器控制、Canvas、定时任务等丰富工具

## 典型使用场景

### 在 Telegram 上使用 AI 助手
通过 OpenClaw，你可以直接在 Telegram 上与 Claude 对话，无需切换应用，让 AI 助手融入你的日常沟通流程。

### 团队协作中的 AI 助手
在 Slack 或 Discord 团队频道中部署 OpenClaw，让团队成员都能方便地使用 AI 助手，提升团队协作效率。

### 语音交互场景
在开车、做饭或其他不方便打字的场景下，通过语音唤醒与 AI 助手自然对话。

### 服务器部署
将 OpenClaw 部署在远程 Ubuntu 服务器上，7×24 小时持续响应，通过 Telegram Bot 随时访问。

## 安全与隐私

OpenClaw 提供多层安全保护：

- **配对机制**：未知发送者需要通过配对码验证
- **本地数据存储**：所有对话数据存储在本地设备
- **Token 认证**：Dashboard 访问需要 Gateway Token 验证
- **权限控制**：细粒度的工具和命令权限管理

<Callout type="warning">
  **安全提示**：OpenClaw 连接到真实的消息平台，请将入站消息视为不可信输入，并合理配置安全策略。
</Callout>

## 技术架构

```
聊天渠道（Telegram/WhatsApp/...）
         ↕
  Gateway 网关（:18789）
         ↕
   AI 模型（Anthropic/OpenAI/...）
```

- **Gateway WebSocket**：单一 WS 控制平面，管理客户端、工具和事件
- **多 Agent 路由**：将入站频道/账户路由到隔离的 Agent
- **插件系统**：通过插件支持各类渠道和工具
- **后台服务**：支持 launchd（macOS）/ systemd（Linux）/ 计划任务（Windows）
