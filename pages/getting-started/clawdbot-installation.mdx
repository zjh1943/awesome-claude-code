# OpenClaw 安装与配置指南

本指南介绍如何安装 OpenClaw 并配置 CC Club 平台的 API 代理。

<Callout type="info">
**关于本文档**
本文档重点帮助你在 OpenClaw 中配置 CC Club 的 `baseUrl` 和 `apiKey`，以便在国内无障碍访问 Claude。对于 OpenClaw 的其他功能和详细说明，请参考[官方文档](https://docs.openclaw.ai)。
</Callout>

## 系统要求

| 项目 | 最低要求 | 推荐 |
|------|----------|------|
| Node.js | ≥ 22.12.0 | 最新 LTS |
| 内存 | 1GB | 4GB+ |
| 操作系统 | macOS 12+ / Ubuntu 20.04+ / Windows 10+ | macOS 14+ |

## 2. 运行引导向导

安装后执行引导，完成首次配置：

```bash
openclaw onboard --install-daemon
```

向导会依次引导你：
1. 选择模型提供商（选 Anthropic）
2. 输入 API 密钥
3. 设置 Gateway 网关（端口、认证方式）
4. 配置消息渠道（Telegram / WhatsApp / Discord 等）
5. 安装后台服务（launchd / systemd / Windows 计划任务）

## 3. 配置 CC Club API 代理

<Callout type="warning">
**重要**：使用 CC Club 需要手动配置两个地方：**API 密钥**和**自定义 baseUrl**。
</Callout>

### 配置文件位置

OpenClaw 的配置目录：

<Tabs items={['macOS / Linux', 'Windows']}>
  <Tabs.Tab>
    ```
    ~/.openclaw/
    ├── openclaw.json               # 主配置文件
    ├── .env                        # 环境变量（API 密钥等敏感信息）
    └── agents/main/agent/
        └── auth-profiles.json      # API Key 凭据
    ```
  </Tabs.Tab>
  <Tabs.Tab>
    ```
    C:\Users\<用户名>\.openclaw\
    ├── openclaw.json               # 主配置文件
    └── agents\main\agent\
        └── auth-profiles.json      # API Key 凭据
    ```
  </Tabs.Tab>
</Tabs>

### 第一步：写入 API Key

编辑 `~/.openclaw/agents/main/agent/auth-profiles.json`（Windows：`C:\Users\<用户名>\.openclaw\agents\main\agent\auth-profiles.json`）：

```json
{
  "version": 1,
  "profiles": {
    "anthropic:default": {
      "type": "api_key",
      "provider": "anthropic",
      "key": "你的CC-Club-API-Key"
    }
  },
  "lastGood": {
    "anthropic": "anthropic:default"
  }
}
```

也可以通过命令行方式配置：
```bash
openclaw onboard --auth-choice apiKey --anthropic-api-key 你的CC-Club-API-Key
```

### 第二步：配置自定义 baseUrl

编辑 `~/.openclaw/openclaw.json`，添加 `models` 节：

```json
{
  "models": {
    "mode": "merge",
    "providers": {
      "anthropic": {
        "baseUrl": "https://claude-code.club/api",
        "models": []
      }
    }
  }
}
```

<Callout type="info">
**`"models": []` 为必填项**，留空表示使用 provider 默认模型列表；省略该字段会导致配置校验报错：
```
Invalid config: models.providers.anthropic.models: expected array, received undefined
```
</Callout>

### CC Club API 节点

根据你的网络情况选择最佳节点：

| 地址 | 说明 |
|------|------|
| `https://claude-code.club/api` | 默认地址（直连延迟 < 100ms 时最快） |
| `https://jp.claude-code.club/api` | 日本节点（阿里云，回国优化） |
| `https://hk.claude-code.club/api` | 香港节点（阿里云，回国优化） |
| `https://sz.ai-code.club/api` | 深圳节点（国内，由服务器代为出境） |

将上述地址填入 `openclaw.json` 的 `models.providers.anthropic.baseUrl`。

### 第三步：重启 Gateway

<Tabs items={['macOS', 'Linux / Ubuntu', 'Windows']}>
  <Tabs.Tab>
    ```bash
    launchctl stop ai.openclaw.gateway
    launchctl start ai.openclaw.gateway
    ```
    或：
    ```bash
    kill $(lsof -ti :18789) && openclaw gateway
    ```
  </Tabs.Tab>
  <Tabs.Tab>
    ```bash
    systemctl --user restart openclaw-gateway.service
    ```
  </Tabs.Tab>
  <Tabs.Tab>
    ```bash
    openclaw gateway stop
    openclaw gateway start
    ```
  </Tabs.Tab>
</Tabs>

## 4. 完整配置示例

以下是一个使用 CC Club 的完整 `openclaw.json` 示例：

```json
{
  "meta": {
    "lastTouchedVersion": "2026.2.19"
  },
  "agents": {
    "defaults": {
      "model": {
        "primary": "anthropic/claude-sonnet-4-6"
      },
      "workspace": "~/.openclaw/workspace",
      "compaction": {
        "mode": "safeguard"
      }
    }
  },
  "models": {
    "mode": "merge",
    "providers": {
      "anthropic": {
        "baseUrl": "https://claude-code.club/api",
        "models": []
      }
    }
  },
  "commands": {
    "native": "auto",
    "nativeSkills": "auto"
  },
  "gateway": {
    "mode": "local",
    "auth": {
      "mode": "token",
      "token": "你的gateway-token"
    },
    "port": 18789
  }
}
```

**关键配置说明：**

| 配置项 | 说明 | 示例值 |
|--------|------|--------|
| `agents.defaults.model.primary` | 默认使用的模型，需加 provider 前缀 | `anthropic/claude-sonnet-4-6` |
| `models.providers.anthropic.baseUrl` | CC Club API 地址 | `https://claude-code.club/api` |
| `models.providers.anthropic.models` | 模型列表，留空使用默认 | `[]` |
| `gateway.auth.token` | Dashboard 访问令牌 | 用 `openssl rand -hex 32` 生成 |
| `gateway.port` | Gateway 监听端口 | `18789` |

## 7. 配置消息渠道

### Telegram

<Steps>
### 创建 Bot

打开 Telegram，找 [@BotFather](https://t.me/BotFather)，发送 `/newbot`，按提示设置名称和用户名，获得 Bot Token（格式：`123456789:ABCDEF...`）。

### 配置渠道

在 `~/.openclaw/openclaw.json` 中添加：

```json
{
  "channels": {
    "telegram": {
      "enabled": true,
      "dmPolicy": "open",
      "allowFrom": ["*"],
      "groupPolicy": "allowlist",
      "streamMode": "partial"
    }
  },
  "plugins": {
    "entries": {
      "telegram": { "enabled": true }
    }
  }
}
```

### 设置 Bot Token

```bash
# ~/.openclaw/.env
TELEGRAM_BOT_TOKEN=123456789:ABCDEF...
```

### Telegram 配对

在 Telegram 给你的 Bot 发消息，Bot 会返回配对码，然后在服务器上批准：

```bash
openclaw pairing approve telegram XXXXXXXX
```
</Steps>

### Discord / Slack / WhatsApp

WhatsApp 需要扫码登录：
```bash
openclaw channels login --channel whatsapp
```

Discord 和 Slack 在 `.env` 中配置 Bot Token，详见[官方文档](https://docs.openclaw.ai)。

## 9. 常见问题

### API Key 报错：`No API key found for provider "anthropic"`

**检查步骤：**
1. 确认文件存在：`~/.openclaw/agents/main/agent/auth-profiles.json`
2. 确认格式正确（参考第 3 节）
3. 重启 Gateway

### 配置文件校验失败

```
Invalid config: models.providers.anthropic.models: expected array, received undefined
```

**解决**：在 `openclaw.json` 的 `models.providers.anthropic` 下添加 `"models": []`。

### Gateway 端口被占用

```
Port 18789 is already in use. pid xxxxx
```

**解决（Linux/macOS）：**
```bash
kill $(lsof -ti :18789)
```

**解决（Windows）：**
```powershell
Stop-Process -Id <pid> -Force
```

### TUI 报 `pairing required` 或 `device token mismatch`

```bash
# 清除旧的设备配对文件
rm -f ~/.openclaw/devices/paired.json \
      ~/.openclaw/identity/device-auth.json \
      ~/.openclaw/identity/device.json

# 重启 Gateway
systemctl --user restart openclaw-gateway.service

# 验证状态
sleep 5 && openclaw gateway status
```

### Dashboard 显示 "unauthorized"

用带 Token 的 URL 访问：
```
http://localhost:18789/#token=你的gateway-token
```

或查看当前 Token：
```bash
openclaw config get gateway.auth.token
```

### 验证代理可达性

```bash
curl -X POST "https://claude-code.club/api/v1/messages" \
  -H "x-api-key: 你的API-Key" \
  -H "anthropic-version: 2023-06-01" \
  -H "content-type: application/json" \
  -d '{"model":"claude-sonnet-4-6","max_tokens":64,"messages":[{"role":"user","content":"hi"}]}'
```

### 更新 OpenClaw

```bash
# 稳定版（推荐）
openclaw update --channel stable

# 或直接 npm 更新
npm update -g openclaw
```

## 与 Claude Code 的区别

| 对比项 | Claude Code | OpenClaw |
|--------|-------------|----------|
| **用途** | 命令行编程助手 | 自托管多渠道 AI 网关 |
| **配置工具** | CC Switch / 环境变量 | 手动编辑 JSON |
| **配置文件** | `~/.claude/settings.json` | `~/.openclaw/openclaw.json` |
| **API Key 存储** | 环境变量 | `auth-profiles.json` |
| **界面** | 命令行 | Web Dashboard |
| **消息平台** | 终端 | Telegram、WhatsApp 等 20+ |
